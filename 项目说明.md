## 🚀 如何使用

### 1. 本地测试
直接在浏览器中打开 `index.html` 即可预览。

### 2. 本地服务器运行（推荐）
由于浏览器的安全限制，建议使用本地服务器：

**使用Python（如果已安装）:**
```bash
# Python 3
python -m http.server 8000

# 然后访问 http://localhost:8000
```

**使用Live Server（VS Code扩展）:**
1. 安装 Live Server 扩展
2. 右键点击 `index.html`
3. 选择 "Open with Live Server"

### 3. 添加背景音乐（可选）
1. 创建 `assets/audio/` 文件夹
2. 添加名为 `jingle-bells.mp3` 的音乐文件
3. 或修改 `index.html` 中的音频文件路径

### 4. 部署到线上

#### 方式一：GitHub Pages
```bash
# 1. 确保代码已提交
git add .
git commit -m "完成圣诞祝福H5项目"
git push origin main

# 2. 在GitHub仓库设置中启用GitHub Pages
# Settings -> Pages -> Source: main branch
```

#### 方式二：Vercel（推荐）
1. 访问 https://vercel.com
2. 连接GitHub仓库
3. 一键部署
4. 自动获得HTTPS域名

#### 方式三：Netlify
1. 访问 https://netlify.com
2. 拖拽项目文件夹到部署区域
3. 立即上线

## 🎨 自定义修改

### 修改祝福语
编辑 `js/blessings.js` 文件，修改 `BLESSINGS` 对象中的内容。

### 修改场景颜色
编辑 `css/scenes/` 下对应的CSS文件，修改渐变色和装饰效果。

### 修改雪花效果
编辑 `js/snow.js` 文件，调整雪花数量、速度等参数。

## ⚠️ 注意事项

1. **外部依赖（CDN）**
   - qrcode.js - 二维码生成
   - html2canvas.js - 截图功能
   - 需要网络连接才能使用这些功能

2. **音频文件**
   - 音频文件需要用户交互后才能播放（浏览器限制）
   - 如果没有音频文件，音乐功能会失效，但不影响其他功能

3. **链接长度**
   - 自定义祝福语较长时，生成的链接也会较长
   - 可以使用短链接服务进一步缩短

4. **浏览器缓存**
   - 修改后需要强制刷新（Ctrl+F5）来清除缓存

## 🔧 故障排查

### 问题1：页面样式错乱
- 检查CSS文件是否正确加载
- 清除浏览器缓存
- 使用开发者工具查看控制台错误

### 问题2：二维码不显示
- 检查网络连接（CDN加载）
- 查看浏览器控制台是否有错误
- 确认 qrcode.js 已成功加载

### 问题3：音乐无法播放
- 确认 `assets/audio/jingle-bells.mp3` 文件存在
- 浏览器需要用户交互后才能播放音频
- 检查音频文件格式是否支持

### 问题4：展示页面无法显示
- 检查URL参数是否完整
- 尝试重新生成链接
- 查看浏览器控制台错误信息
